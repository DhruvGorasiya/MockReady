## Sprint Goal

A candidate can sign up, start a mock interview session, answer questions generated by the AI, and receive dimension-by-dimension feedback — all persisted to the database.

---

## User Stories

### US-1: Candidate Authentication
**As a** candidate
**I want to** register and log in with email and password
**So that** my interview sessions are saved to my account

**Acceptance criteria:**
- [ ] `POST /api/v1/auth/register` creates a user with bcrypt-hashed password
- [ ] `POST /api/v1/auth/login` returns a signed JWT on valid credentials
- [ ] Invalid credentials return 401 with a generic message (no email/password hint)
- [ ] JWT is validated on every protected route via `get_current_user` dependency
- [ ] Frontend stores token and redirects to dashboard on login

---

### US-2: Session Creation with AI Questions
**As a** candidate
**I want to** start a mock interview session for a specific role and interview type
**So that** I receive relevant questions generated by AI

**Acceptance criteria:**
- [ ] `POST /api/v1/sessions` creates a session and triggers the Question Generation Agent
- [ ] 5 questions are generated and snapshotted into `SessionQuestion` records
- [ ] Session status is `active` after creation
- [ ] Questions are role/type-specific (e.g., "Software Engineer — behavioral")
- [ ] Agent invocation is logged with agentId, latency, model version

---

### US-3: Answer Submission and Evaluation
**As a** candidate
**I want to** submit an answer to a question and receive AI scoring
**So that** I know how well I performed across each dimension

**Acceptance criteria:**
- [ ] `POST /api/v1/sessions/{id}/answers` accepts answer text and question index
- [ ] Answer Evaluation Agent and Feedback Synthesis Agent run concurrently via `asyncio.gather()`
- [ ] Scores are stored in `EvaluationScore` with `ai_score` populated
- [ ] Feedback is stored and linked to the evaluation
- [ ] Response includes scores (1-10) per dimension and feedback text

---

### US-4: Candidate Dashboard
**As a** candidate
**I want to** view my past sessions and scores
**So that** I can track my improvement over time

**Acceptance criteria:**
- [ ] `/dashboard` page lists sessions with date, role, and composite score
- [ ] Clicking a session shows per-question scores and feedback
- [ ] Data loads from `GET /api/v1/sessions` (auth-protected)

---

## Definition of Done

- Code merged to `main` via PR (no direct pushes)
- All acceptance criteria checked off
- Unit tests written before implementation (TDD — red/green/refactor)
- CI pipeline passes (lint, typecheck, tests, security scan)
- No new `any` in TypeScript
- Agent invocations logged
